# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Preferred: use **GitHub Security Advisories** (private report) if enabled for this repository.

Alternative: email `maintainers@meminit.io` with:

- a clear description of the issue and impact,
- minimal reproduction steps or a proof-of-concept,
- affected versions/commits (if known).

If the vulnerability involves secrets, **do not** send real credentials or tokens.

## Supported Versions

Meminit is early-stage. We aim to patch issues on `main` promptly and will cut a
release if needed.
